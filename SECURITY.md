# Security Policy

## Supported Versions

This is a curated resource list (awesome-list). There is no executable code in
this repository, so traditional software vulnerability reporting does not apply.

| Version | Supported |
| --- | --- |
| Latest commit on `main` | Yes |

## Reporting a Vulnerability

Since this repository contains only documentation and curated links, security
issues are limited to:

- **Malicious links**: a listed resource URL has been replaced with a malicious
  or phishing site.
- **Spam or inappropriate content**: a submitted resource contains misleading,
  harmful, or spam content.
- **Compromised dependencies**: a listed tool or library has been compromised
  or hijacked on PyPI/GitHub.

If you discover any of the above, please **do not open a public issue**. Instead,
report it privately:

1. Email the maintainers at: **[INSERT SECURITY EMAIL]**
2. Include the affected section, URL, and a description of the issue.

You will receive a response within **48 hours**. If confirmed, the offending
link will be removed immediately.

## Scope

This policy applies only to the `awesome-behave` repository content. For
security issues in Behave itself or any listed third-party tool, please refer
to their respective security policies.
